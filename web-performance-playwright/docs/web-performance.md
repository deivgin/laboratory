# Web performance

> Web performance is all about making websites fast, including making slow processes seem fast.
>
> -- <cite>MDN</cite>

Web performance goal is to measure the percieved experience of a website.

- Reducing overall load time
- Making the site usable as soon as possible
  - lazy loading
  - time to interactive (TTI) - how long it takes for the site to be interactable after initial load
- Smoothness and interactivity
  - making the app feel smooth
- Percieved performance
- Performance mesurements
  - Actual and persieved speed - objective mesurements and how long it feels

## Minimising performance bottlenecks

- Minimise initial load - download content that is interactable first and load all other content later.
- Optimise assets that are being loaded - images, video, styles - consider using lazy loading.
- Prevent jumping content and reflows - elements should not flow without user interaction.
- Avoid font file delas
- Visible interactive elements have to be always interactable. It should be interactable in 50ms and repianted in 16.67ms
